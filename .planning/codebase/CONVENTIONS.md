# Coding Conventions

**Analysis Date:** 2026-04-10

## Languages & Frameworks

**Primary:**
- TypeScript - Used across all 14 Nuxt layer packages
- Vue 3 - Composition API with `<script setup>` for all UI components
- Python - Used in `packages/python-backend` for AI/RAG services

**TypeScript Configuration:**
- Each package has its own `tsconfig.json` extending from site `.nuxt/tsconfig.json`
- Strict mode enabled via Nuxt's TypeScript configuration

## Naming Patterns

**Files:**
- Vue components: `PascalCase.vue` (e.g., `CsvUploader.vue`, `PostCard.vue`)
- TypeScript services: `camelCase.service.ts` (e.g., `post.service.ts`, `chat.service.ts`)
- TypeScript types/interfaces: `PascalCase` (e.g., `ServiceResponse`, `PostCreateBase`)
- Test files: `*.test.ts` for Vitest, `test_*.py` for pytest

**Functions:**
- camelCase for functions and methods
- Use descriptive verb prefixes: `get`, `create`, `update`, `delete`, `bulk`, `validate`

**Variables:**
- camelCase for local variables and parameters
- Interfaces and types use PascalCase
- Constants use UPPER_SNAKE_CASE

**Directories:**
- Use kebab-case: `server/services/`, `utils/helpers/`, `tests/e2e/`

## Code Style

**Formatting:**
- ESLint configuration via `@nuxt/eslint` module
- 13 packages have `eslint.config.js`, site package uses `eslint.config.mjs`
- ESLint config extends from each package's `.nuxt/eslint.config.mjs` (auto-generated by Nuxt)

**Configuration Pattern:**
```javascript
// packages/bulk-scheduler/eslint.config.js
import withNuxt from './.playground/.nuxt/eslint.config.mjs'
export default withNuxt()
```

```javascript
// packages/site/eslint.config.mjs
// @ts-check
import withNuxt from './.nuxt/eslint.config.mjs'

export default withNuxt(
  // Your custom configs here
)
```

**Vue Components:**
- Use `<script setup lang="ts">` for all new Vue components
- Props defined with `defineProps<{...}>()`
- Events defined with `defineEmits<{...}>()`
- Use JSDoc comments for component documentation

**Component Example:**
```vue
<script lang="ts" setup>
/**
 * Component Description: CSV file uploader
 *
 * @author [Name] <[email]>
 * @version 0.0.1
 */

const props = defineProps<{
  maxFileSize?: number
  acceptedTypes?: string[]
}>()

const emit = defineEmits<{
  fileSelected: [file: File]
  error: [message: string]
}>()
</script>
```

## Service Layer Pattern

**Service Location:** `packages/*/server/services/`

**Service Response Pattern:**
All service methods must return `ServiceResponse<T>`, never throw exceptions:
```typescript
// packages/db/server/services/types.ts
export interface ServiceResponse<T = any> {
  data?: T
  error?: string
  code?: string
}
```

**Service Structure:**
```typescript
// packages/db/server/services/post.service.ts
export class PostService {
  private db = useDrizzle()

  async create(userId: string, data: PostCreateBase): Promise<ServiceResponse<Post>> {
    try {
      this.validateCreateData(data)
      
      const id = crypto.randomUUID()
      const now = dayjs.utc().toDate()

      const [post] = await this.db.insert(posts).values({
        id,
        userId,
        businessId: data.businessId,
        content: data.content,
        mediaAssets: data.mediaAssets ? JSON.stringify(data.mediaAssets) : null,
        scheduledAt: dayjs(data.scheduledAt).utc().toDate(),
        targetPlatforms: JSON.stringify(data.targetPlatforms),
        status: 'pending',
        createdAt: now,
        updatedAt: now
      }).returning()

      return { data: post }
    } catch (error) {
      if (error instanceof ValidationError) {
        return { error: error.message, code: error.code }
      }
      return { error: 'Failed to create post: ' + error }
    }
  }
}
```

## Error Handling

**Custom Error Classes:**
```typescript
// packages/db/server/services/types.ts
export class ServiceError extends Error {
  constructor(
    message: string,
    public code: string,
    public statusCode: number = 500
  ) {
    super(message)
    this.name = 'ServiceError'
  }
}

export class ValidationError extends ServiceError {
  constructor(message: string, public field?: string) {
    super(message, 'VALIDATION_ERROR', 400)
    this.name = 'ValidationError'
  }
}

export class NotFoundError extends ServiceError {
  constructor(resource: string) {
    super(`${resource} not found`, 'NOT_FOUND', 404)
    this.name = 'NotFoundError'
  }
}

export class UnauthorizedError extends ServiceError {
  constructor(message: string = 'Unauthorized') {
    super(message, 'UNAUTHORIZED', 401)
    this.name = 'UnauthorizedError'
  }
}
```

**Rules:**
- Never throw exceptions from service methods
- Always return `ServiceResponse<T>` with either `data` or `error`
- Catch ValidationError separately to return proper error code
- Use custom error classes for different error types

## Comments & Documentation

**When to Comment:**
- JSDoc for component definitions
- Author attribution: `@author Name <email>`
- Version info: `@version x.x.x`
- Parameter descriptions in JSDoc

**Example:**
```typescript
/**
 * Creates a new post for the specified user
 * @param userId - The unique identifier of the user
 * @param data - Post creation data
 * @returns ServiceResponse with created post or error
 */
async create(userId: string, data: PostCreateBase): Promise<ServiceResponse<Post>>
```

## Module Design

**Barrel Files:**
- Use index exports for service directories
- Export services as named exports

**Export Pattern:**
```typescript
// packages/db/server/services/post.service.ts
export class PostService { ... }

// packages/db/server/services/index.ts
export { PostService } from './post.service'
export { ChatService } from './chat.service'
```

**Database Schema:**
- Schema files in `packages/db/db/schema/`
- Use Drizzle ORM with `drizzle-orm`
